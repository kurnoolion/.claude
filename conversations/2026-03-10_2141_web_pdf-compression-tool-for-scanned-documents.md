**Summary:** **Conversation Overview**

The person requested help building a Python command-line tool to compress PDF files specifically generated from scanned book pages processed through Tesseract OCR. The tool needed two compression modes: compressing to a target percentage of the original file size, and compressing to under a specific file size (e.g., 10MB, 500KB). Claude built the program using a binary search algorithm on JPEG quality settings, with a DPI-reduction fallback for cases where quality adjustment alone cannot reach the target. The tool always saves output as a new file (e.g., `<original>_compressed.pdf`) and includes an explicit safety check preventing the output path from matching the input path.

The person asked a clarifying question confirming the program does not overwrite the input file, which Claude confirmed and explained with reference to the specific safety check in the code. Subsequently, the person encountered runtime errors when attempting to run the program on Windows. The errors stemmed from the original implementation's dependency on `pdf2image`, which requires Poppler — an external C++ tool not bundled with Python and not straightforwardly available on Windows. Claude rewrote the script to replace `pdf2image` and Poppler entirely with PyMuPDF (`fitz`), a self-contained Python wheel that requires no external system dependencies and installs via pip on Windows, macOS, and Linux. The revised script requires only three pip packages: `pymupdf`, `Pillow`, and `pypdf`. Key CLI options include `-p` for percentage-based compression, `-s` for size-based compression, `--dpi` to control rasterisation resolution (default 200), and `--grayscale` for additional size reduction on black-and-white scans.

Here's your PDF compressor! It's tailored specifically for scanned/image-based PDFs (the kind Tesseract produces).

**How it works:** It rasterises each PDF page, then uses a **binary search on JPEG quality** to find the best quality setting that hits your target — so it never wastes quality unnecessarily.

---

**Requirements** (all likely already installed):
```bash
pip install Pillow pdf2image pypdf
# also needs: poppler-utils (for pdf2image)
# sudo apt install poppler-utils   OR   brew install poppler
```

---

**Usage examples:**

| Goal | Command |
|---|---|
| Compress to 50% of original size | `python pdf_compressor.py scan.pdf -p 50` |
| Compress to under 10 MB | `python pdf_compressor.py scan.pdf -s 10MB` |
| Compress to under 500 KB | `python pdf_compressor.py scan.pdf -s 500KB` |
| Custom output filename | `python pdf_compressor.py scan.pdf -s 5MB -o small.pdf` |
| Higher quality raster (less compression) | `python pdf_compressor.py scan.pdf -p 40 --dpi 300` |
| Black & white scan (extra savings) | `python pdf_compressor.py scan.pdf -s 10MB --grayscale` |

**Tips:**
- `--dpi 200` (default) is the sweet spot for readable text. Drop to `--dpi 150` or `100` if you need to squeeze harder.
- `--grayscale` can cut size by an additional ~50% for colour scans of black-ink books.
- The output is always saved as `<original>_compressed.pdf` unless you use `-o`.

No, it **never overwrites your input file**. The program always creates a new file. By default it saves the compressed version as `<original_name>_compressed.pdf` in the same folder. For example:

- Input: `scan.pdf` → Output: `scan_compressed.pdf`
- Input: `my_book.pdf` → Output: `my_book_compressed.pdf`

There's even an explicit safety check in the code that throws an error if you accidentally point `-o` to the same path as the input:

```python
if os.path.abspath(input_pdf) == os.path.abspath(output_pdf):

```

Your original file is always left untouched.
